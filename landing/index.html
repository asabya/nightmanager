Orchestration without the noise.

Subagents keeps complex implementation work legible by assigning each phase to a focused role instead of one overloaded assistant.

F

Finder

Searches the codebase, identifies ownership boundaries, and returns the smallest useful map before changes begin.

O

Oracle

Provides deeper architectural judgment for risky choices, ambiguous requirements, and trade-off heavy tasks.

M

Manager

Turns context into a concrete plan, preserves constraints, and keeps implementation focused on the objective.

W

Worker

Makes the smallest viable code change, verifies it, and reports concise evidence for review.

Install

Install as a Pi extension.

Use subagents from this repo while developing, or install the local package into Pi once it is ready.

Install locally

Register the local subagents project with Pi from an absolute path.

local extension 
pi install /absolute/path/to/subagents

Run from repo

During development, execute the TypeScript entrypoint directly from the repository.

development 
pi -e ./index.ts

Built usage

Build once, then point Pi at the compiled JavaScript output.

compiled 
npm run build
pi -e ./dist/index.js

Finder, Oracle, Manager, and Worker become available as Pi tools after install.
